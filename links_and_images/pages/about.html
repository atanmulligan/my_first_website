About Commit Messages

Commit Messages are very important. Employers will look to see if you can write them well, and having a good commit message history lets you collaborate better and fix bugs faster. It also gives your brain a jog when you come back to a project that you haven't done in a while.

Differentiating bad and good commits

Bad commits are vague and unprecise. Good commit messages consist of a subject and a body You must describe the problem, and then solve it.

EG: Added missing links and added a description to logo. Readers wont read images to people with accesibility issues.

Think of commits as a "snapshot" of the code you just uploaded or changed. This version will be saved so you can look back at it in the future. Make sure to commit every time you make a meaningful change to your code. In fact, now that I am about to add this page, I need to add a commit message as well.
