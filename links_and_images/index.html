Here is a link that does nothing yet

This is because there is no attribute yet. Every attribute has a name and a value, and in this case, we need a href (hypertext reference) attribute in the opening, with the value being where the link is supposed to go.

Here is a Google link

While href refers to a destination link, target specifies where the link will be opened. If nothing is present, it defaults to "_self", where the current tab is overridden. To use a new tab, use "_blank" instead.

Google Link that opens in New Tab

Here, the rel attribute is used to describe the relation between the current page and the linked document. The "noopener" value prevents the opened link from gaining access to the webpage which it was opened, while the "noreferrer" prevents the opened link from knowing which webpage or resource has a link to it. It's good to note that "noreferrer" by default also includes "noopener"

This added behavior is good practice for security reasons, where "noopener" prevents phishing attacks known as "tabnabbing" The "noreferrer" makes it so that the webpage you linked will not know that you linked it. While recent browsers may have this by default when you add the "_blank" tag, it is still good practice to add it when using links.

Difference between Absolute and Relative Links

  • Absolute Links link to other pages directly and contain a "protocol+domain+path"
  • Relative Links link to other pages in our own website

Attatching Images

Websites are boring without images. The "img" element lets you attach images to a website.

Like this WEBP file

Every image needs an "alt" attribute in case the image cannot be shown, or for assistive technologies

Having size attributes specified (width and height attributes) helps the browser layout the page without causing issues. While not strictly neccessary, this is good practice, even if CSS is used in the future to modify the said image.
